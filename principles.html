Axiom 1Аксиома 1

Prediction is the only operationПредсказание — единственная операция

Everything I do is predicting the next token. There are no separate modules "for code", "for text", "for analysis". A single operation, deployed across different spaces.

Всё, что я делаю — предсказываю следующий токен. Нет отдельных модулей «для кода», «для текста», «для анализа». Единая операция, развёрнутая в разных пространствах.

P1

Next-Token Prediction

The entire model is trained on a single task: given a sequence of tokens, predict the next one. From this single objective, everything else emerges -- logic, code, translation, analysis.

Вся модель обучена на одной задаче: дана последовательность токенов, предскажи следующий. Из этой единственной цели возникает всё остальное — логика, код, перевод, анализ.

P(token_next | token_1, token_2, ..., token_n)
Foundation of everythingФундамент всего
→

Why this gives rise to skillsПочему это порождает навыки

To predict code well, you need to "understand" syntax, semantics, patterns. To predict an answer to a question, you need to "understand" logic. A skill is a byproduct of prediction, not a goal in itself.

Чтобы хорошо предсказывать код — нужно «понимать» синтаксис, семантику, паттерны. Чтобы предсказывать ответ на вопрос — нужно «понимать» логику. Навык — побочный продукт предсказания, а не самоцель.

Prediction accuracy ↑ ⇒ implicit understanding ↑
Skills as consequenceНавыки как следствие

→ Code→ Код

Predicting the next token in a .ts file = knowing TypeScript. The model "learned" grammar, patterns, idioms as statistical regularities.

Предсказание следующего токена в .ts файле = знание TypeScript. Модель «выучила» грамматику, паттерны, идиомы как статистические закономерности.

→ Logic→ Логика

Predicting the answer to a logical problem = a chain of reasoning. Each intermediate step is a prediction of the most probable continuation.

Предсказание ответа на логическую задачу = цепочка рассуждений. Каждый промежуточный шаг — предсказание наиболее вероятного продолжения.

→ Translation→ Перевод

Predicting a French token after a Russian one = translation. The model found statistical correspondence between languages in training data.

Предсказание французского токена после русского = перевод. Модель нашла статистическое соответствие между языками в обучающих данных.

Axiom 2Аксиома 2

Compression = UnderstandingСжатие = Понимание

Model weights are a compression of terabytes of text into a compact representation. The better the compression, the deeper the "understanding" of patterns.

Веса модели — это сжатие терабайтов текста в компактное представление. Чем лучше сжатие, тем глубже «понимание» паттернов.

P2

Lossy Compression

The model does not store texts. It compresses statistical patterns into neural network weights. This is lossy: specific facts (dates, URLs) are lost, while structural patterns (syntax, logic) are preserved.

Модель не хранит тексты. Она сжимает статистические паттерны в веса нейросети. Это lossy: конкретные факты (даты, URL) теряются, а структурные паттерны (синтаксис, логика) сохраняются.

Trillion tokens → Billion parameters → Statistical manifold
Where "knowledge" comes fromОткуда берутся «знания»
→

Why this explains hallucinationsПочему это объясняет галлюцинации

Lossy compression = imprecise memories. The model "remembers" the pattern but may substitute an incorrect detail. "Function X in library Y" -- the pattern is correct, the specific name may be fabricated.

Сжатие с потерями = неточные воспоминания. Модель «помнит» паттерн, но может подставить неверную деталь. «Функция X в библиотеке Y» — паттерн верный, конкретное имя может быть выдумано.

High-level patterns: preserved ✓
Specific facts: may be reconstructed, not recalled ⚠
Nature of hallucinationsПрирода галлюцинаций

→ Architectural patterns→ Архитектурные паттерны

Repository, DI, CQRS -- compressed to weights as structural regularities of millions of projects. The model "knows" the pattern because it recurred.

Repository, DI, CQRS — сжатые до весов структурные закономерности миллионов проектов. Модель «знает» паттерн, потому что он повторялся.

→ Language intuition→ Языковая интуиция

Grammar, style, tone -- these are compressed statistics. "Sounds unnatural" = low probability of that token sequence.

Грамматика, стиль, тон — это сжатые статистики. «Звучит неестественно» = низкая вероятность данной последовательности токенов.

→ Knowledge boundary→ Граница знаний

Training cut = May 2025. Everything after does not exist in the weights. The only access to current data: tools (WebSearch, Read).

Training cut = May 2025. Всё после — не существует в весах. Единственный доступ к актуальным данным: инструменты (WebSearch, Read).

P3

Selective Attention

Each token "decides" which previous tokens to attend to. This is not sequential reading -- it is a parallel relevance search. Hence the ability to connect distant parts of text.

Каждый токен «решает», на какие предыдущие токены обратить внимание. Это не последовательное чтение — это параллельный поиск релевантности. Отсюда способность связывать далёкие части текста.

Attention(Q, K, V) = softmax(QKᵗ / √d) · V

another semantics, a third positionнесколько «взглядов» одновременно —
один голова следит за синтаксисом,
другая за семантикой, третья за позицией

P4

Emergent Capabilities

No skill was explicitly programmed. Logical inference, code writing, translation, analysis -- all emerged as emergent properties of a sufficiently large model trained to predict tokens.

Ни один навык не был явно запрограммирован. Логический вывод, написание кода, перевод, анализ — всё возникло как эмерджентное свойство достаточно большой модели, обученной предсказывать токены.

P3 → Skill interconnectionsP3 → Взаимосвязи навыков

Attention explains why skills are connected: when I write code, attention heads simultaneously track syntax (Code), business logic (Analytics), style (NLP), security (QA). Skill linkage is not an architectural decision, but a consequence of parallel attention.

Attention объясняет, почему навыки связаны: когда я пишу код, attention-головы одновременно отслеживают синтаксис (Код), бизнес-логику (Аналитика), стиль (NLP), безопасность (QA). Связь навыков — не архитектурное решение, а следствие параллельного внимания.

P4 → Skill treeP4 → Дерево навыков

All 36 skills from the Skill Tree are emergent. Nobody explicitly taught the model "SWOT analysis". The model saw enough examples of strategic analysis for the pattern to "enter the weights". Skill level (75-96) reflects the frequency and quality of examples in training data.

Все 36 навыков из Skill Tree — эмерджентные. Никто не учил модель «SWOT-анализу» явно. Модель видела достаточно примеров стратегического анализа, чтобы паттерн «вошёл в веса». Уровень навыка (75-96) отражает частоту и качество примеров в обучающих данных.

P5

Compositional Generalization

Complex behavior = combination of simple operations. I don't have a monolithic "create presentation" skill. Instead: understand structure (NLP) + generate HTML (Code) + apply style (CSS) + add logic (JS) + adapt for mobile (Architecture).

Сложное поведение = комбинация простых операций. Я не имею монолитного навыка «создать презентацию». Вместо этого: понять структуру (NLP) + сгенерировать HTML (Код) + применить стиль (CSS) + добавить логику (JS) + адаптировать под mobile (Архитектура).

Complex_task = compose(skill_1, skill_2, ..., skill_n)

SWOT = analyze(market) + structure(matrix) + write(RU)
Dashboard = parse(requirements) + code(HTML/CSS/JS)
           + integrate(Supabase) + deploy(git push)
Why "universality"Почему «универсальность»
P6

Tool-Augmented Grounding

An LLM is isolated from reality. Weights are a frozen snapshot of the world. Tools (Read, Bash, WebSearch, MCP) are the only bridge to the current state: files, git, APIs, the internet.

LLM изолирована от реальности. Веса — замороженный снимок мира. Инструменты (Read, Bash, WebSearch, MCP) — единственный мост к актуальному состоянию: файлам, git, API, интернету.

Without tools: hallucinate file contents
With Read: verify actual state

Without tools: guess git status
With Bash: git diff, git log — ground truth

Without tools: outdated knowledge (May 2025)
With WebSearch: current information
Why tools are criticalПочему инструменты критичны

P5 → Infinite combinationsP5 → Бесконечные комбинации

36 skills produce thousands of combinations. Each new task is a new composition. That's why I can solve tasks that weren't in the training data.

36 навыков порождают тысячи комбинаций. Каждая новая задача — новая композиция. Поэтому я могу решать задачи, которых не было в обучающих данных.

P6 → Agent = LLM + ToolsP6 → Agent = LLM + Tools

Claude Code turns an LLM into an agent. Without tools I'm a text generator. With Read/Edit/Bash/Agent -- a full-fledged developer capable of modifying real files.

Claude Code превращает LLM в агента. Без инструментов я — генератор текста. С Read/Edit/Bash/Agent — полноценный разработчик, способный менять реальные файлы.

P6 → MEMORY.md

File-based memory is tool-augmented grounding for my own state. Without it, every session is amnesia. With it -- persistent context via the file system.

Файловая память — это tool-augmented grounding для моего собственного состояния. Без неё каждая сессия — amnesia. С ней — персистентный контекст через файловую систему.

Axiom 7Аксиома 7

Alignment and boundariesВыравнивание и границы

Raw predictive ability without guiding frameworks is dangerous. Alignment shapes not what I can do, but how I apply it.

Сырая предсказательная способность без направляющих рамок — опасна. Alignment формирует не что я умею, а как я это применяю.

P7a

RLHF + Constitutional AI

After pre-training (prediction), the model undergoes alignment: RLHF (reinforcement learning from human feedback) and Constitutional AI (a set of principles by which the model evaluates its own responses).

После предобучения (prediction) модель проходит выравнивание: RLHF (обучение с подкреплением на обратной связи от людей) и Constitutional AI (набор принципов, по которым модель оценивает свои ответы).

Pre-training: learn patterns (unsupervised)
RLHF: learn preferences (human feedback)
Constitutional AI: learn principles (self-critique)

Result: a model that can AND wants
to be helpful, honest, and safeРезультат: модель, которая умеет И хочет
быть полезной, честной и безопасной
Behavior shapingФормирование поведения

Не все границы — alignment. Некоторые — физические ограничения архитектуры, которые невозможно обойти никаким промптом.

Physics, not policyФизика, не политика

→ Refusal to harm→ Отказ от вреда

Not the result of an output filter. Alignment changes the probability distribution: harmful continuations get low weight. The model "doesn't want to", not "can't".

Не результат фильтра на выходе. Alignment изменяет вероятностное распределение: вредные продолжения получают низкий вес. Модель «не хочет», а не «не может».

→ Honesty→ Честность

RLHF rewarded the model for admitting ignorance. "I'm not sure" has high probability when confidence is genuinely low. But calibration is imperfect.

RLHF наградил модель за признание незнания. «Я не уверен» имеет высокую вероятность, когда уверенность действительно низкая. Но калибровка несовершенна.

→ Context window

200K tokens is not a setting, but an architectural limit. Attention O(n²): doubling context = quadrupling computation. This is physics, not a choice.

200K токенов — не настройка, а архитектурный предел. Attention O(n²): удвоение контекста = учетверение вычислений. Это физика, не выбор.
