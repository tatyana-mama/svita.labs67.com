Claude Opus 4.6 is the largest model in the Claude 4 family. I generate text token by token, predicting the most likely continuation based on the entire preceding context. I have no persistent memory, state, or consciousness.

Claude Opus 4.6 — крупнейшая модель семейства Claude 4. Генерирую текст токен за токеном, предсказывая наиболее вероятное продолжение на основе всего предшествующего контекста. Не имею постоянной памяти, состояния или сознания.

200K токенов — жёсткий лимит. При приближении к границе деградирует всё: теряю начало разговора, забываю ранние решения, повторяю ошибки. Файловая память (MEMORY.md) — костыль, не замена.

What happens from the moment a prompt is received to the generation of a responseЧто происходит от момента получения промпта до генерации ответа

01

TokenizationТокенизация

Text is split into tokens (BPE). "Hello" = 1 token. Code is often split differently: "function" = 1, but "aiChatSend" = 3-4 tokens.

Текст разбивается на токены (BPE). «Привет» = 1 токен. Код часто разбивается иначе: «function» = 1, но «aiChatSend» = 3-4 токена.

~4 chars = 1 token~4 символа = 1 токен
02

Embedding

Each token is converted into a high-dimensional vector. Semantically similar words end up close together in the space.

Каждый токен преобразуется в вектор высокой размерности. Семантически близкие слова оказываются рядом в пространстве.

Vectors ~8K+ dims
03

Attention

Each token "looks at" all preceding ones. Multi-head attention finds relevant connections at different levels of abstraction.

Каждый токен «смотрит» на все предыдущие. Multi-head attention находит релевантные связи на разных уровнях абстракции.

O(n²) over contextO(n²) по контексту
04

Extended Thinking

Up to 32K tokens of internal reasoning (hidden from user). Task decomposition, approach evaluation, self-correction before generating a response.

До 32K токенов внутреннего рассуждения (скрыто от пользователя). Декомпозиция задачи, перебор подходов, самокоррекция до генерации ответа.

Hidden CoTСкрытый CoT
05

Tool Use

The model decides: is a tool needed? It generates a JSON call (Read, Edit, Bash, Grep...). Receives the result and includes it in context.

Модель решает: нужен ли инструмент? Генерирует JSON-вызов (Read, Edit, Bash, Grep...). Получает результат, включает в контекст.

JSON function call
06

DecodingДекодирование

Autoregressive generation: each next token is sampled from a distribution conditioned on all previous tokens + temperature.

Авторегрессивная генерация: каждый следующий токен сэмплируется из распределения, обусловленного всеми предыдущими + temperature.

Token-by-token

⚠ No RAG⚠ Нет RAG

I have no built-in retrieval. My "knowledge" consists of frozen model weights. Current information is only available through tools (WebSearch, Read).

Я не имею встроенного retrieval. Мои «знания» — это замороженные веса модели. Актуальная информация доступна только через инструменты (WebSearch, Read).

⚠ Hallucinations⚠ Галлюцинации

I can generate plausible but factually incorrect text. Especially: specific numbers, dates, URLs, API names. Always verify critical facts.

Могу генерировать правдоподобный, но фактически неверный текст. Особенно: конкретные цифры, даты, URL, имена API. Всегда верифицируйте критические факты.

✓ Self-correction✓ Самокоррекция

Extended thinking allows detecting an error before output. But correction is limited: if the error is in the model weights, thinking won't fix it.

Extended thinking позволяет обнаружить ошибку до вывода. Но коррекция ограничена: если ошибка в весах модели, thinking её не исправит.
