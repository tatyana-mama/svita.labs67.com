Processing Pipeline

Пайплайн обработки

What happens from the moment a prompt is received to the generation of a response

Что происходит от момента получения промпта до генерации ответа

01

Tokenization

Токенизация

Text is split into tokens (BPE). "Hello" = 1 token. Code splits differently.

Текст разбивается на токены (BPE). «Привет» = 1 токен. Код разбивается иначе.

~4 chars = 1 token ~4 символа = 1 токен
02

Embedding

Each token becomes a high-dimensional vector. Semantically similar words end up close together.

Каждый токен преобразуется в вектор высокой размерности. Семантически близкие слова оказываются рядом.

Vectors ~8K+ dims
03

Attention

Each token "looks at" all preceding ones. Multi-head attention finds relevant connections at different abstraction levels.

Каждый токен «смотрит» на все предыдущие. Multi-head attention находит связи на разных уровнях абстракции.

O(n²) over context O(n²) по контексту
04

Extended Thinking

Up to 32K tokens of internal reasoning (hidden). Task decomposition, self-correction before response.

До 32K токенов внутреннего рассуждения (скрыто). Декомпозиция задачи, самокоррекция до ответа.

Hidden CoT Скрытый CoT
05

Tool Use

The model decides: is a tool needed? Generates a JSON call (Read, Edit, Bash, Grep...). Gets result into context.

Модель решает: нужен ли инструмент? Генерирует JSON-вызов (Read, Edit, Bash, Grep...). Результат в контекст.

JSON function call
06

Decoding

Декодирование

Autoregressive generation: each next token sampled from a distribution conditioned on all previous + temperature.

Авторегрессивная генерация: каждый следующий токен сэмплируется из распределения всех предыдущих + temperature.

Token-by-token

⚠ No RAG

⚠ Нет RAG

I have no built-in retrieval. My "knowledge" consists of frozen model weights. Current information is only available through tools (WebSearch, Read).

Я не имею встроенного retrieval. Мои «знания» — это замороженные веса модели. Актуальная информация доступна только через инструменты (WebSearch, Read).

⚠ Hallucinations

⚠ Галлюцинации

I can generate plausible but factually incorrect text. Especially: specific numbers, dates, URLs, API names. Always verify critical facts.

Могу генерировать правдоподобный, но фактически неверный текст. Особенно: конкретные цифры, даты, URL, имена API. Всегда верифицируйте критические факты.

✓ Self-correction

✓ Самокоррекция

Extended thinking allows detecting an error before output. But correction is limited: if the error is in the model weights, thinking won't fix it.

Extended thinking позволяет обнаружить ошибку до вывода. Но коррекция ограничена: если ошибка в весах модели, thinking её не исправит.
